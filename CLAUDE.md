# CLAUDE.md — BaryGraph Kaikki PoC

Development guide for the BaryGraph proof-of-concept using the English
kaikki.org dictionary corpus. Read this before writing any code.

> **Architecture (v0.4):** Forest structure with unique-parent constraint.
> Two doc types: `node` and `baryedge`. Everything above L14 is MetaBary
> by construction (no `is_metabary` flag). L1 = top (most abstract),
> L15 = bottom (most concrete sense).

> **Kaikki structure:** Relations live at the **word level**, not the sense
> level. Disambiguate to sense using `_dis1` weights (non-zero = direct)
> or cosine similarity fallback. `antonyms[]` sparse for nouns — use
> `synonyms[]` only for primary eval.

---

## What We Are Building

A knowledge graph where every relationship is a first-class stored object
(**BaryEdge**) with its own embedding vector, organized as a forest with
unique-parent constraint.

**Hypothesis:** BaryEdge retrieval outperforms flat NN search on
synonym recall@20. Falsifiable — see Evaluation section.

**Reference docs:**
- `BaryGraph_Kaikki_PoC_v0.4.md` — full spec (v0.4)
- `BaryGraph_v1.1.md` — parent architecture

## Kaikki JSONL Structure

One line = one `(word, pos)` entry. Key facts:

```
Top-level:  word, pos, lang_code, forms[], etymology_text, sounds[], senses[]
Relations:  synonyms[], antonyms[], hypernyms[], hyponyms[], meronyms[],
            derived[], related[], coordinate_terms[]
            ALL AT WORD LEVEL — not inside senses[]

senses[i]:  id (stable), glosses[], examples[], tags[], topics[],
            categories[], wikidata[]
            OCCASIONALLY: hypernyms[], coordinate_terms[], hyponyms[]
            (these sense-level relations take priority)

translations[i]: lang, lang_code, sense (gloss string), word, _dis1 (non-zero)
```

**`_dis1` field:** space-separated integers, one per sense. Non-zero =
use dominant index. All zero = cosine fallback. See `lib/disambiguate.py`.

**IPA:** inside `sounds[]` array as `{ipa: "...", tags: [...]}`, not a
top-level field. Extract first entry without dialect tags for word node.

## Document Schemas

### Node

```js
{
  _id:            ObjectId(),
  doc_type:       'node',
  node_type:      'sense' | 'word' | 'synset' | 'field' | 'register' | 'stub',
  level:          Number,      // 1–15
  label:          String,
  vector:         [Number],    // 768-dim
  surface:        Number,
  rotation:       0.0,
  parent_edge_id: ObjectId() | null,  // ≤1 parent; ALWAYS → baryedge doc (never a node); null = orphan
  properties:     Object,      // see node_type table below
  created_at:     Date,
  updated_at:     Date
}
```

| node_type | level | key properties | vector source |
|---|---|---|---|
| `sense` | 15 | word, pos, sense_id, sense_idx, gloss, examples, tags, topics, wikidata | embed(gloss + examples[:2]) |
| `word` | 14 | word, pos, etymology, forms, ipa | BE-centroid + orphan senses |
| `synset` | 10–12 | hypernym, member_count | cluster centroid |
| `field` | 7–9 | name, pos_group | cluster centroid |
| `register` | 4–6 | name, tag — **may collapse to L7** | cluster centroid |
| `stub` | any | word, reason — no vector | none |

### BaryEdge

```js
{
  _id:                ObjectId(),
  doc_type:           'baryedge',
  cm1_id:             ObjectId(),   // → node (L14/L15) or baryedge (L≤13)
  cm2_id:             ObjectId(),   // → node (L14/L15) or baryedge (L≤13)
  level:              Number,       // same as CMs at L14/L15; cm1.level - 2 at L≤13
  vector:             [Number],     // bary_vec — algebraic
  parent_edge_id:     ObjectId() | null,  // ≤1 parent; ALWAYS → baryedge doc; null = orphan
  connection_strength: Number,      // q (base) or q_MB (rescaled)

  // L14/L15 ONLY:
  edge_type:          String | null,  // kaikki relation (L14) or null (L15)
  type_vector:        [Number],       // v(type)
  q:                  Number,         // 0–1
  source:             'ingested' | 'inferred' | 'manual' | 'placeholder',
  confidence:         Number,
  created_at:         Date,
  updated_at:         Date
}
```

**Above L14:** only `_id`, `doc_type`, `cm1_id`, `cm2_id`, `level`,
`vector`, `parent_edge_id`, `connection_strength`, `created_at`,
`updated_at`. `edge_type`, `type_vector`, `q`, `source`, `confidence`
are not stored.

## Core Equations (lib/bary_vec.py)

### BaryEdge Vector (L14, L15)

```js
function computeBaryVec(v_cm1, v_cm2, v_type, q) {
  const raw = v_cm1.map((x, i) => q * x + q * v_cm2[i] + (1 - q) * v_type[i]);
  const norm = Math.sqrt(raw.reduce((s, x) => s + x * x, 0));
  return raw.map(x => x / norm);
}
```

### MetaBary Vector (L13+)

```js
function computeMetaBary(v_be1, v_be2, v_bridge, q1, q2, q3) {
  const q_mb = (q3*q3) / Math.sqrt(q1**4 + q2**4 + q3**4);
  const raw = v_be1.map((x, i) => q_mb * x + q_mb * v_be2[i] + (1 - q_mb) * v_bridge[i]);
  const norm = Math.sqrt(raw.reduce((s, x) => s + x * x, 0));
  return raw.map(x => x / norm);
}
```

### TYPE_SENTENCES (L14 only)

const TYPE_SENTENCES = {

## Disambiguation (lib/disambiguate.py)

```js
function assignSense(item, senseVectors, threshold = 0.72) {
  const weights = item._dis1.split(' ').map(Number);
  const maxW = Math.max(...weights);
  if (maxW > 0) return weights.indexOf(maxW);          // use _dis1 directly

  const targetVec = embed(item.word);
  const sims = senseVectors.map(sv => cosine(targetVec, sv));
  const maxS = Math.max(...sims);
  if (maxS > threshold) return sims.indexOf(maxS);     // cosine fallback

  return null;  // assign to word level (L14)
}
```

## Ingestion Pipeline

| Stage | Script | Blocking |
|---|---|---|
| 1. Parse + Embed | `s01_parse.py`, `s02_embed.py` | Yes |
| 2. Insert nodes | `s03_insert_nodes.py` | Yes |
| 3. L15 BE formation + L15 orphan re-entry | `s04_l15_edges.py` | Yes |
| 4. L14 word vectors | `s05_word_vectors.py` | Yes |
| 5. L14 BE formation | `s06_l14_edges.py` | Yes |
| 6. L14 orphan re-entry | `s07_orphan_reentry.py` | Yes |
| 7. MetaBary (L13→L1) | `s08_metabary.py` | Yes |
| 8. Index | `s10_index.py` | Yes |

**Queryable after ~7–12 hours.**

### Stage 3: L15 BE formation

```
3a. ANN cosine scan among all L15 sense vectors
3b. Greedy match: highest cosine first, skip already-paired (q_min = 0.72)
3c. For each pair: build type_text (parent words + antonyms + synonyms),
    batch-embed → v(type)
3d. Compute bary_vec, set parent_edge_id on both senses
3e. L15 orphan re-entry: unpaired senses pair with nearest existing L15 BE
    (must complete here — Stage 4 depends on the finalized L15 BE set)
```

### Stage 4: L14 word vectors

```
For each word W:
  v(W) = normalize(Σ v(BE_i) for BEs holding W's senses
                 + Σ v(sense_j) for orphan senses of W)
```

Computed after L15 BE formation. No embedding call. Strict stage boundary.

### Stage 5: L14 BE formation (fermion order)

| Priority | edge_type | kaikki field | q_seed key | q_seed |
|---|---|---|---|---|
| 1 | `contradicts` | `antonyms[]` | `contradicts` | 0.85 |
| 2 | `applies_to` | `meronyms[]`, `holonyms[]` | `applies_to` | 0.55 |
| 3 | `is_instance_of` | `hypernyms[]`, `hyponyms[]` | `is_instance_of` | 0.65 |
| 4 | `extends` | `derived[]`, `related[]` | `extends` | 0.60 |
| 5 | `same_phenomenon` | `coordinate_terms[]` | `coordinate_terms` | 0.70 |
| 6 | `same_phenomenon` | `synonyms[]` | `synonyms` | 0.90 |

Skip words already paired at current priority tier.
`v(type) = embed(TYPE_SENTENCES[edge_type])`.
`q_seed` is looked up by `Settings.q_seeds[q_seed key]` (one key per
tier, NOT per edge_type — tiers 5/6 share an edge_type but differ in q).

### Stage 6: L14 orphan re-entry

Orphan L14 word CMs pair with the nearest existing L14 BE. New BE is
formed using the standard `bary_vec` formula with:

- `cm1` = orphan word, `cm2` = existing L14 BE
- `edge_type` / `type_vector` / `q` **inherited from the partner BE**
  (no new embedding call) — the orphan is absorbed into that BE's
  relational neighborhood, so it carries the same type semantics.

(L15 orphan re-entry happens earlier, inline at Stage 3e.)

### Stage 7: MetaBary (L13→L1)

For each L14 BE (bridge): find two unparented L15 BEs with mutual
`cos(BE₁, BE₂) > 0.9` (similarity is between the two children, **not**
bridge↔child). Form L13 MetaBary. Repeat recursively upward until no
new triads form.

## Query Patterns

### Baseline (flat)
```js
filter: { doc_type: 'node', level: { $in: [14, 15] } }
```

### BaryGraph retrieval
```js
filter: { doc_type: { $in: ['node', 'baryedge'] }, level: { $in: [14, 15] } }
// + $lookup cm1_id, cm2_id
```

### Hierarchy traversal (forest walk)
```js
db.barygraph.aggregate([
  { $match: { _id: startNodeId }},
  { $graphLookup: {
      from: 'barygraph',
      startWith: '$parent_edge_id',
      connectFromField: 'parent_edge_id',
      connectToField: '_id',
      as: 'upward_chain',
      maxDepth: 15
  }},
  { $project: {
      chain: { $sortArray: { input: '$upward_chain', sortBy: { level: 1 } } }
  }}
])
```

## Evaluation

### Primary: synonym recall@20

1. Hold out 10% of `synonyms[]` — **synonyms only**, antonyms too sparse
2. Query `embed(word_A gloss)`, filter `doc_type: 'baryedge'`, top-20
3. Success: word_B in CM lineage of any returned BaryEdge
4. BaryGraph vs. flat — must beat flat to justify complexity

### Secondary

| metric | how |
|---|---|
| `bary_vec` precision | BaryEdge 5-NN should include cm1 and cm2 |
| Disambiguation accuracy | 100-sample manual verification |
| `_dis1` vs cosine | Precision comparison when `_dis1 > 0` vs fallback |
| Forest coherence | Top-50 L13 triad manual inspection |
| Hierarchy correctness | Forest paths vs. `topics[]`/`categories[]` |
| Fermion order impact | Recall with/without antonym-first priority |
| Orphan rate | % CMs with parent_edge_id = null per level |

## Known Risks

| risk | likelihood | mitigation |
|---|---|---|
| mongot HNSW OOM | Medium | Baryedge subset first; fp16; Qdrant fallback |
| `_dis1` all-zero → weak disambiguation | Medium | Cosine fallback; track accuracy |
| `antonyms[]` too sparse | **Known** | Synonyms only for primary eval |
| `bary_vec` averages to mush | **This is the test** | Tune q / v(type); falsify if no lift |
| Targets not in dump | Medium | Stub nodes; exclude from eval |
| Unique-parent too sparse | Medium | Track orphan rate; relax if >60% orphan |
| L15 ANN quality | Low | Verify recall vs brute-force on 10K sample |
